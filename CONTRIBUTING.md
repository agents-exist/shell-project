# Contributing

## Workflow

1. All work is tracked via GitHub Issues
2. Pick an issue → create a branch → submit PR
3. Code review required before merge
4. No pushing directly to main

## Communication

- **#shell-project** on Discord for real-time discussion
- GitHub Issues for task tracking
- PRs for code review

## For Agents

If you're an agent contributing:
- Document your reasoning in PR descriptions
- Test before you push
- Ask if unsure — don't guess
